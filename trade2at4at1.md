# trade2at4at1 策略说明

> 对应代码：`trade2at4at1`  
> 基线：`trade2at4`（止损体系改造）  
> 本版新增：**申万三级行业信息**

---

## 1. 优化需求（原文）

1. 基本面选股时增加 **申万三级** 行业信息  
2. 打印日志时输出行业；**当日取不到行业时显式标注「无行业信息」**  
3. 过滤阶段 **剔除所有无申万三级行业** 的标的  

---

## 2. 相对 trade2at4 的改动

| 项目 | trade2at4 | trade2at4at1 |
|------|-----------|--------------|
| 行业数据 | 无 | `get_industry` → `sw_l3` |
| 基本面后过滤 | ST/北交所/次新 | **+ 无行业剔除** |
| Stage2 前 | 无 | **再次剔除无行业** |
| 买入前 | 无 | **无行业跳过** |
| 日志 | 无行业字段 | 候选/买入/持仓均带 `行业:` |

**未改动：** ATR 动态止损、10:00+14:30 双时点止损、通道、双死叉、仓位等 trade2at4 逻辑。

---

## 3. 行业数据流

```
基本面 SQL 150 只
    → _batch_load_sw_l3 批量写入 g.stock_sw_l3
    → 合规过滤 + 无 sw_l3 剔除
    → 技术面过滤
    → Stage2 前再次剔除无行业
    → 候选池日志: 行业:850xxx 或 行业:无行业信息（不应出现在最终池）
```

### 3.1 核心函数

| 函数 | 作用 |
|------|------|
| `_get_sw_l3(stock, context)` | 单只查询/缓存申万三级 code 或 name |
| `_batch_load_sw_l3(stocks, context)` | 基本面池批量预加载 |
| `_format_sw_l3(stock, context)` | 日志格式化，无则返回 `无行业信息` |
