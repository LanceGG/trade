# trade2at4at5 策略说明

> 对应代码：`trade2at4at5`  
> 基线：`trade2at4`（ATR 动态止损 + 10:00/14:30 双检）  
> 本版新增：**昨/今涨跌幅动量加分排序**

---

## 1. 优化需求（原文）

对于所有筛选出来的股票，昨日涨跌幅和今日涨跌幅计算因子都是 1，即涨一个点分数 +1，跌一个点分数 -1，然后按分数排序。

---

## 2. 相对 trade2at4 的改动

| 项目 | trade2at4 | trade2at4at5 |
|------|-----------|--------------|
| 候选排序 | 仅形态基础分（100/85/80） | **基础分 + 昨动量 + 今动量** |
| 昨涨跌幅权重 | — | `momentum_factor_yesterday = 1` |
| 今涨跌幅权重 | — | `momentum_factor_today = 1` |
| 07:30 选股 | 形态分排序 | 基础分 + **昨日动量**（今日暂为 0） |
| 10:00 买入 | 按 07:30 顺序 | **刷新今日动量后重排**再买入 |

**未改动：** ATR 动态止损、14:30 复检、通道 3 日、双死叉、仓位管理等 trade2at4 逻辑。

---

## 3. 计分公式

```
昨日动量 = (昨收 / 前收 - 1) × 100 × 因子1    # 涨1% → +1分
今日动量 = (现价 / 昨收 - 1) × 100 × 因子2    # 涨1% → +1分
总分     = 形态基础分 + 昨日动量 + 今日动量
```

| 形态 | 基础分 |
|------|--------|
| wedge_pop | 100 |
| ema_crossback | 85 |
| base_break | 80 |

### 示例

| 股票 | 形态 | 基础 | 昨+3% | 今+1% | 总分 |
|------|------|------|-------|-------|------|
| A | ema_crossback | 85 | +3 | +1 | **89** |
| B | wedge_pop | 100 | -2 | -4 | **94** |

B 形态更好但动量弱，A 动量强则可能反超（取决于具体数值）。

### 扣分说明

涨跌幅为**有符号**百分点：涨 1% → +1 分，**跌 1% → -1 分**（公式对称，负涨跌幅自动扣分）。

```
昨收下跌3% → yesterday_mom = -3
10:00 当日跌2% → today_mom = -2
ema_crossback 基础85 → 总分 = 85 + (-3) + (-2) = 80
```

日志用 `:+.1f` 显示正负号。07:30 / 10:00 各输出 **`动量统计`**：昨日扣分只数、今日扣分只数、合计动量负分只数及区间。

---

## 4. 核心函数

| 函数 | 作用 |
|------|------|
| `_get_yesterday_return_pct` | `attribute_history` 取昨/前收，算昨日涨跌幅（百分点） |
| `_get_today_return_pct` | 现价（或开盘价）相对昨收，算今日涨跌幅 |
| `_get_pre_close` | 取昨收（聚宽无 `pre_close` 字段） |
| `_apply_momentum_score` | 基础分 + 昨/今动量（跌为负） |
| `_log_momentum_stats` | 汇总昨日/今日扣分只数与动量区间 |
| `_refresh_today_stocks_momentum` | 10:00 刷新今日动量并重排 `g.today_stocks` |

---

## 5. 数据流

```
Stage2 形态通过
    → 07:30: 总分 = 基础分 + 昨日动量（今日=0）
    → 按总分排序 → g.today_stocks
    → 10:00: 刷新今日动量，重排
    → 按新顺序买入
```

**说明：** 07:30 盘前尚无成交价，今日动量在 10:00 买入前才纳入排序，与「涨跌幅」语义一致。

---

### 扣分说明

涨跌幅为**有符号**百分点，跌则动量为负、从总分中扣除：

```
昨收下跌3% → yesterday_mom = -3
10:00 当日跌2% → today_mom = -2
ema_crossback 基础85 → 总分 = 85 + (-3) + (-2) = 80
```

日志用 `:+.1f` 显示符号（如 `昨动量:-3.0`）。  
07:30 / 10:00 各输出一行 **`动量统计`**：昨日扣分只数、今日扣分只数、合计动量负分只数及区间。

## 7. 设计意图与风险

**意图：** 在形态合格的前提下，优先买入**短期动量更强**的标的（昨+今均涨得分高），弱化「形态好但正在下跌」的候选。

**风险：**

- 动量与 trade2at4at4「大跌过滤」方向相反——at5 加分追涨，at4 剔除大跌；不宜盲目叠加
- 涨跌幅加分会抬高高 beta 股排序，可能加剧 CANSLIM 高波动池的止损频率
- 需与 `trade2at4` 基线回测对比：是否提升胜率、是否增加追高回撤
