# trade2at4at2 策略说明

> 对应代码：`trade2at4at2`  
> 基线：`trade2at4`（ATR 动态止损 + 10:00/14:30 双检）  
> 本版新增：**昨日大跌过滤 + 买入当日跌幅过滤 + MA200 距离加分**

---

## 1. 优化需求（原文）

1. 筛选股票时，过滤**昨天大跌超过 5%** 的股票  
2. 买入时，若股票**当日下跌超过 3%** 则不买入，继续试下一只  
3. Stage2 形态检测时，**距离 MA200 越近，得分越高**

---

## 2. 相对 trade2at4 的改动

| 项目 | trade2at4 | trade2at4at2 |
|------|-----------|--------------|
| 技术面过滤 | 52周回撤/MA200/价格/成交额 | **+ 昨日跌幅 >5% 剔除** |
| 10:00 买入 | 涨停/ST 等常规过滤 | **+ 当日跌幅 >3% 跳过，试下一只** |
| Stage2 得分 | 形态固定分（100/85/80） | **+ MA200 距离加分（最多 +15）** |
| 参数 | — | `yesterday_drop_filter=0.05`、`intraday_drop_skip=0.03`、`ma200_score_bonus_max=15` |

**未改动：** ATR 动态止损、14:30 止损复检、通道 3 日、双死叉、仓位管理等 trade2at4 逻辑。

---

## 3. 逻辑说明

### 3.1 昨日大跌过滤（08:30 选股）

在 `get_canslim_pool` 技术面循环中，对已通过 MA200 等检查的标的：

```
yesterday_ret = close[-1] / close[-2] - 1
若 yesterday_ret < -5% → 剔除
```

- `close[-1]`：前一交易日收盘（即「昨天」）  
- `close[-2]`：再前一交易日收盘  
- 统计写入 `stats['yesterday_drop_excluded']`，日志输出剔除数量  

### 3.2 买入当日跌幅过滤（10:00 交易）

在 `market_open_trade` 候选股筛选中，通过 `_get_pre_close`（`attribute_history` 取昨收）计算：

```
intraday_ret = (last_price - 昨收) / 昨收
若 intraday_ret <= -3% → 跳过该只，继续遍历 g.today_stocks 下一只
```

- 被跳过的股票**不计入** `candidates`，槽位留给后续候选  
- 日志示例：`600519 当日跌幅-4.2%超过3%，跳过买入，试下一只`

### 3.3 MA200 距离加分（Stage2 排序）

新增 `_apply_ma200_score_bonus(base_score, close, ma200)`：

```
dist = close / ma200 - 1          # 相对 MA200 的溢价比例
bonus = (0.30 - dist) / 0.30 * 15 # dist 越小（越贴近 MA200）bonus 越高
final_score = min(100, base_score + bonus)
```

| 距 MA200 | 加分（base=85 时） |
|----------|-------------------|
| +5% | +12.5 → 97 |
| +15% | +7.5 → 92 |
| +30% 及以上 | +0 → 85 |

形态基础分不变：`wedge_pop=100`、`ema_crossback=85`、`base_break=80`。

## 4. 数据流

```
基本面 SQL 150 只
    → ST/北交所/次新过滤
    → 技术面（52周/MA200/价格/成交额）
    → 【新增】昨日跌幅 >5% 剔除
    → detect_cycle_stage + MA200 距离加分
    → 按最终得分排序 → g.today_stocks
    → 10:00 买入前 【新增】当日跌幅 >3% 跳过
```
